## 一、背景：v4 GP 引擎的核心问题

| 问题 | 表现 | 影响 |
|------|------|------|
| 模块全局变量 `_TREE_GEN_CONFIG` | 跨模块 `from...import` 赋值不穿透 | TreeGenConfig 不生效，debug 困难 |
| 无类型安全 | `and`/`or` 在连续值子树上生成 `HIGH and HIGH` | 大量无效表达式，浪费算力 |
| 无模式区分 | 因子发现（连续值）和信号发现（布尔值）混用 | 搜索方向混乱 |
| 子类覆盖 `run()` 脆弱 | ProgressGPEngine 漏同步 globals | 配置无声失效 |
| 无自变量适应 | 静态权重 20 代不变 | 好方向不能强化，差方向不能淘汰 |

## 二、v5 核心设计：三层自适应体系

```
┌─ 方向层（v5 核心创新）─────────────────────
│  控制「往哪搜」— 树生成概率自适应
│  ├─ 变量权重      → AMOUNT 高频出现就自动提权
│  ├─ 函数权重      → ts_corr 出 SR 就自动提权
│  ├─ 结构类型权重  → binary_op（乘积）出效果就自动提权
│  └─ 多实例自然分化 → 同一初始偏置，不同 seed 各自收敛
│
├─ 算子层（AW-MEP / DGEP）───────────────────
│  控制「怎么变」— 遗传算子自适应
│  ├─ 变异率自适应  → 停滞时自动加码
│  ├─ 交叉率自适应  → 收敛时自动降温
│  └─ 算子选择自适应 → 哪个算子有效就多用
│
└─ 评估层（KB01 方法论）─────────────────────
   控制「选哪个」— 多方向并行验证
   └─ 多实例并行 → 各自收敛 → 人验证分化方向
```

## 三、核心模块设计

### 3.1 GPTreeConfig — 纯数据配置

```python
@dataclass
class GPTreeConfig:
    """树生成配置 — 实例绑定，无全局变量"""
    
    # 模式
    mode: str = 'continuous'   # continuous | predicate | hybrid
    
    # 深度
    max_depth: int = 4
    
    # 权重（None = 等概率）
    var_weights: dict = None       # {'AMOUNT': 3, 'VOLUME': 2}
    ts_weights: dict = None        # {'ts_corr': 3, 'ts_cov': 3}
    math_weights: dict = None      # {'neg': 3, 'cos': 1.5}
    feature_weights: dict = None   # {'feature_1arg': 0.5}
    group_weights: dict = None     # mode 自动过滤不适用组
    
    # 动态权重
    adaptive: bool = False
    adaptive_lr: float = 0.3        # EMA 更新速率
    adaptive_every: int = 3         # 每 N 代更新
    adaptive_floor: float = 0.05    # 权重地板
```

**mode 自动过滤规则**：
- `continuous`: 禁用 comparison/logic/ternary（只出连续值）
- `predicate`: 禁用 ts/math/feature（只出布尔表达式）
- `hybrid`: 全部开放

### 3.3 GPEngine — 遗传算法引擎

```python
class GPEngine:
    """遗传算法引擎 — 配置自包含，进度回调替代子类"""
    
    def __init__(self, data, fitness_calc,
                 config: dict,              # 种群/代数/变异权重
                 tree_config: GPTreeConfig = None,
                 seed_exprs: list = None):
        self.tree_gen = GPTreeGen(tree_config or GPTreeConfig())
        self.fitness_calc = fitness_calc
        self.cfg = config
        # 算子自适应状态
        self._stagnation_counter = 0
        self._last_best = -999
    
    def run(self, callback=None) -> 'GPEngine':
        """
        callback(gen, best_fitness, best_expr, stats) → None
        替代 ProgressGPEngine 子类模式
        """
        for gen in range(self.generations):
            self._evaluate_population()
            
            # 进度回调
            if callback:
                callback(gen, best.fitness, best.expression_str, stats)
            
            # 方向层 — 动态权重更新
            if self.tree_gen.cfg.adaptive and gen % adaptive_every == 0:
                self.tree_gen.update_weights(self._top(10))
            
            # 算子层 — 变异率/交叉率自适应
            self._adapt_operators(gen)
            
            if gen < self.generations - 1:
                self.population = self._next_generation(gen)
    
    def _adapt_operators(self, gen):
        """AW-MEP 风格：根据停滞检测自适应调整算子权重"""
        if self._stagnation_counter > 3:
            self.mutation_prob = min(0.5, self.mutation_prob * 1.3)
            self.crossover_prob = max(0.2, self.crossover_prob * 0.8)
        else:
            self.mutation_prob = max(0.15, self.mutation_prob * 0.95)
```

## 四、应用层（kb03/05-a 使用方式）

```python
# 1. 定义初始偏置
config = GPTreeConfig(
    mode='continuous',
    max_depth=4,
    adaptive=True,
    adaptive_lr=0.3,
    adaptive_every=3,
    var_weights={'AMOUNT': 3, 'VOLUME': 2, 'CLOSE': 1},
    ts_weights={'ts_corr': 3, 'ts_cov': 3, 'ts_roc': 2},
    math_weights={'neg': 3, 'cos': 1.5, 'gauss': 1.5},
)

# 2. 多实例并行探索（各自收敛到不同方向）
for seed in [42, 123, 456]:
    engine = GPEngine(
        data=raw_ohlcv,
        fitness_calc=MaxAbsFitness(),
        config={'population_size': 200, 'generations': 20},
        tree_config=config,
        seed_exprs=seeds,
        random_seed=seed,
    )
    
    # 进度回调（替代 ProgressGPEngine 子类）
    def on_gen(gen, fitness, expr, stats):
        print(f"Gen {gen:3d} | fit={fitness:+.4f} | {expr[:60]}")
    
    engine.run(callback=on_gen)
    
    # 观察收敛方向
    final_weights = engine.tree_gen.cfg.var_weights
    # seed=42 → AMOUNT=5.2, ts_corr=4.8  （量价相关性方向）
    # seed=123 → VOLUME=6.1, gauss=3.2  （量能钟形方向）
    # seed=456 → OPEN=4.3, cos=5.1, ts_delta=3.8  （开盘变换方向！）
```

## 五、与业内方案对比

| 维度 | WS-GP | AlphaForge | AW-MEP | **v5** |
|------|-------|------------|--------|--------|
| 搜索空间 | 单模板锁定 | DL 全空间 | GP 全空间 | **GP 全空间 + 权重引导** |
| 自适应类型 | ❌ | 组合权重 | 算子率 | **树生成概率 + 算子率** |
| 新方向发现 | ❌ | ✅ | ✅ | ✅ **多实例自动分化** |
| 初始化 | 已知因子 | 随机 | 随机 | **裸变量 + 偏置** |
| 可解释性 | ★★★ | ★ | ★★ | ★★★ |
| 人机分工 | 人给模板 | 全自动 | 全自动 | **人给偏置 → GP 自适应** |

**v5 的独特定位**：在「可解释 GP × 方向自适应」的交叉点上无直接竞品。

## 七、关键发现记录（来自 KB01 + 05-a）

1. **cos = 量纲敏感哈希**：窄范围内等价钟形，跨 2π 周期后产生量化哈希
2. **decay 是关键结构**：裸 cos=0.56 → decay10=0.98（+75%增量）
3. **深度线性链 GP 弱**：decay(cos(ΔP)) 需 4 节点精确匹配，GP 概率低
4. **不同信号源 × 不同变换 → 天然正交**：7 因子相关系数中位数 0.015
5. **31 行业截面天花板 ≈ 1.3~1.4**：两条独立路径收敛到同一区间
6. **纯动量/反转/K线/月频在 31 行业全部无效**
7. **and/or 对连续值无意义**：`HIGH and HIGH ≡ HIGH`
8. **from...import 赋值不穿透模块**：必须用 `module.attr = value`
