## 二、P0 (Critical) — 必须立即修复

### P0-1: `_ts_rank_core` 当前值为 NaN 时返回"过期排名"而非 NaN

**文件:行号**: [functions.py:369-370](file:///d:/01-Doc/Quant/ft2/utils/ast/functions.py#L369-L370)

**问题描述**:
```python
sorted_v = np.sort(valid)
pos = np.searchsorted(sorted_v, valid[cnt - 1])  # ← 用 valid[cnt-1] 而非 x[i]
r[i] = (pos + 1) / cnt
```
`ts_rank` 的语义是"当前值 `x[i]` 在窗口内的排名"。当 `x[i]` 为 NaN（停牌、缺失）时，`valid[cnt-1]` 是窗口内**最后一个有效值**（某个 `x[j]`, `j < i`），而非 `x[i]`。代码把过去某天的排名当作今天的排名写入 `r[i]`。

**影响分析**:
- `ts_rank` 是因子表达式最常用原语之一（如 `cs_rank(ts_rank(ts_roc(CLOSE, 5), 60))`）。
- A 股数据普遍存在停牌（CLOSE=NaN），`ts_roc(CLOSE,5)` 在停牌日返回 NaN，传入 `ts_rank` 后触发此 bug。
- 返回的"过期排名"是 `[1/cnt, 1]` 区间的有效浮点数，会被下游 `cs_rank` 当作真实值参与截面排名，**静默污染截面排序**：停牌股票可能占据 Top-N 名额，挤掉本应入选的活跃股票。
- 这不是前瞻偏差，而是"数据陈旧"——用历史排名冒充当前排名，且无任何 NaN 标记可供下游识别。

**对比**: `_ts_zscore_core`([functions.py:649](file:///d:/01-Doc/Quant/ft2/utils/ast/functions.py#L649)) 正确使用 `x[i]`；`_ts_scale_core`([functions.py:555](file:///d:/01-Doc/Quant/ft2/utils/ast/functions.py#L555)) 也正确使用 `x[i]`。`_ts_rank_core` 与它们不一致。

**修复方案**:
```python
if np.isnan(x[i]):
    continue
# ...
pos = np.searchsorted(sorted_v, x[i])  # 用 x[i] 而非 valid[cnt-1]
r[i] = (pos + 1) / cnt
```

**修复状态**: ✅ 已修复 (2026-07-06)

## 三、P1 (High) — 影响结果正确性

### P1-1: `_ts_quantile_core` nearest-rank 索引 off-by-one

**文件:行号**: [functions.py:623](file:///d:/01-Doc/Quant/ft2/utils/ast/functions.py#L623)

**问题描述**:
```python
idx = min(int(p * cnt), cnt - 1)   # int() = floor
r[i] = sorted_v[idx]
```
docstring 声明"WQ nearest-rank"。标准 nearest-rank 方法定义为：`ordinal_rank = ceil(p * N)`（1-indexed），即 0-indexed 的 `ceil(p*N) - 1`。代码用 `floor(p * cnt)`，当 `p * cnt` 为整数时与标准差 1。

**数值验证** (p=0.5, N=10):
| 方法 | idx (0-based) | 取值 |
|------|:---:|:---:|
| 标准 nearest-rank (ceil) | ceil(5)-1 = **4** | 第5小值 |
| 代码 (floor) | floor(5) = **5** | 第6小值 |
| numpy linear (原版) | (4+5)/2 插值 | 中间值 |

bug 仅在 `p*N` 为整数时触发（偶数窗口 + p=0.5/0.8 等，恰是常见场景）。

**影响分析**:
- 作为分位数**阈值**使用时（如 `CLOSE > ts_quantile(CLOSE, 20, 0.8)`），阈值取错一个位置，导致选股边界错误。
- 作为 `cs_rank` 输入时，所有股票同向偏移，相对排名影响较小，但仍非正确实现。

**修复方案**:
```python
idx = int(np.ceil(p * cnt)) - 1
if idx < 0: idx = 0
elif idx > cnt - 1: idx = cnt - 1
r[i] = sorted_v[idx]
```

**修复状态**: ✅ 已修复 (2026-07-06)

### P1-2: `ts_resid` / `ts_slope` / `ts_rsq` / `ts_intercept` / `ts_predict` 在 `window<3` 时返回 0.0 而非 NaN（忽略 fill_value）

**文件:行号**: [functions.py:711-712](file:///d:/01-Doc/Quant/ft2/utils/ast/functions.py#L711-L712)

**问题描述**:
```python
def _ts_linear_reg_core(x, window, rettype, fill_value):
    n = len(x)
    if window < 3:
        return np.zeros(n)        # ← 无视 fill_value, 硬编码 0.0
    r = np.full(n, fill_value)    # ← fill_value 仅在此处生效
```
`ts_slope`/`ts_rsq`/`ts_intercept`/`ts_predict` 传入 `fill_value=np.nan`，但 `window<3` 分支直接 `return np.zeros(n)`，**完全绕过 fill_value**。只有 `ts_resid`（fill_value=0.0）巧合一致。

**影响分析**:
- `ts_slope(x, 2)` 返回全 0 斜率，意为"无趋势"——这是**虚假信号**。正确语义应为 NaN（数据不足，无法计算）。
- 若因子表达式误用小窗口（如 `ts_slope(CLOSE, 2)`），全 0 会被 `cs_rank` 当作真实值排名，所有股票并列同分，静默丢失全部信息。
- `ts_rsq(x, 2)` 返回全 0（"完全无拟合"）同样是误导性信号。

**修复方案**:
```python
if window < 3:
    return np.full(n, fill_value)  # 尊重 fill_value
```

**修复状态**: ✅ 已修复 (2026-07-06)

### P1-3: `_ts_linear_reg_core` (ts_resid/ts_predict) 当前值为 NaN 时返回过期残差/预测

**文件:行号**: [functions.py:747-751](file:///d:/01-Doc/Quant/ft2/utils/ast/functions.py#L747-L751)

**问题描述**:
```python

    predicted = a + b * (cnt - 1)
    r[i] = valid[cnt - 1] - predicted   # ← 用 valid[cnt-1] 而非 x[i]
```

```python

    continue
# ...

    predicted = a + b * (window - 1)

elif rettype == 3:
```

### P1-4: `ts_resid` 冷启动期返回 0.0 而非 NaN，违反冷启动保护原则

**文件:行号**: [functions.py:965](file:///d:/01-Doc/Quant/ft2/utils/ast/functions.py#L965), [functions.py:713](file:///d:/01-Doc/Quant/ft2/utils/ast/functions.py#L713)

**问题描述**:
```python
def ts_resid(x, window):
    return _ts_linear_reg_core(np.asarray(x, float), int(window), 2, 0.0)
                                                              # ↑ fill_value=0.0
```
文件头 docstring([functions.py:52](file:///d:/01-Doc/Quant/ft2/utils/ast/functions.py#L52)) 明确声明"冷启动保护：前 window-1 天返回 NaN"。但 `ts_resid` 的 `fill_value=0.0` 使冷启动期返回 0.0。残差为 0 意为"价格恰等于趋势预测"，是**虚假的中性信号**。

- 若各股票数据起始日不同（A 股新股陆续上市），某日部分股票处于冷启动期（返回 0.0），部分已有效（返回真实残差），0.0 会混入 `cs_rank` 排名，把"数据不足"误判为"残差居中"。

**修复方案**:
```python
def ts_resid(x, window):
    return _ts_linear_reg_core(np.asarray(x, float), int(window), 2, np.nan)
```

**修复状态**: ✅ 已修复 (2026-07-06)

**文件:行号**: [functions.py:720-737](file:///d:/01-Doc/Quant/ft2/utils/ast/functions.py#L720-L737)

### P2-3: `_ts_skew_core` / `_ts_kurt_core` 使用混合公式，偏离标准偏度/峰度

**文件:行号**: [functions.py:500-505](file:///d:/01-Doc/Quant/ft2/utils/ast/functions.py#L500-L505), [functions.py:532-537](file:///d:/01-Doc/Quant/ft2/utils/ast/functions.py#L532-L537)

**问题描述**:
```python
z = (x[j] - mean) / std      # std = 样本标准差 (ddof=1)
m3 += z * z * z
r[i] = m3 / cnt              # ← 分母用 cnt (总体均值), std 用 (cnt-1) (样本)
```
分子 `m3/cnt` 用总体均值，分母 `std` 用样本（ddof=1）。这是非标准混合。与 scipy `skew(bias=True)`（总体偏度）的比值 = `((n-1)/n)^1.5`：n=20 时偏低 7.4%，n=60 时偏低 2.5%。峰度同理比值为 `((n-1)/n)^2`。

**影响分析**:
- 对**排名类因子**（`cs_rank(ts_skew(...))`）：同窗口下所有股票同比例缩放，相对排名不变，影响小。
- 对**阈值类用法**（`ts_skew(...) > 0.5`）：绝对值偏差导致阈值判断错误。
- docstring 声称"对齐 WQ 规范"，但混合公式非任何标准定义。

**修复建议**: 明确选择一种：
- 总体偏度（对齐 scipy bias=True）：`std` 用 `sqrt(ss/cnt)`（ddof=0）。
- 样本偏度（对齐 scipy bias=False）：加偏度修正系数 `sqrt(n*(n-1))/(n-2)`。

**修复状态**: ⏳ 待处理

## 二、P1 (High) — 影响结果正确性

### P1-A: `_feature_hv` 用 `np.std` 不跳过 NaN，停牌期波动率全 NaN

**文件:行号**: [functions.py:1340-1345](file:///d:/01-Doc/Quant/ft2/utils/ast/functions.py#L1340-L1345)

**问题描述**:
```python
def _feature_hv(close, period=20):
    c = np.asarray(close, float)
    rets = np.diff(c) / np.where(c[:-1] > 0, c[:-1], 1)
    rets = np.insert(rets, 0, 0)
    return _rolling(rets, period, lambda a: np.std(a, ddof=1) * np.sqrt(252) * 100)
```

`_rolling` 注释（[functions.py:178-179](file:///d:/01-Doc/Quant/ft2/utils/ast/functions.py#L178-L179)）明确约定："func 应使用 nan 安全版本 (np.nanmean, np.nanmax 等)"。但这里用 `np.std` 而非 `np.nanstd`，窗口内任一 NaN 使整个窗口返回 NaN。

**影响分析**:
- A 股普遍存在停牌（CLOSE=NaN），`np.diff` 传播 NaN 到 rets。
- 停牌后 `period` 天内波动率全 NaN（即使停牌只 1 天，后续 19 天也全 NaN）。
- 与同族 ts_std（用 numba 跳过 NaN）语义不一致：`hv(CLOSE, 20) ≠ ts_std(RETURNS, 20) * sqrt(252) * 100`，前者更"严苛"。
- 下游若用 `cs_rank(hv(CLOSE, 20))`，停牌股票得 NaN，被 `_cs_rank_core` 跳过，但相邻活跃股票因停牌日的 NaN 残留导致窗口不足也变 NaN，**级联放大缺失范围**。

**修复方案**:
```python
return _rolling(rets, period, lambda a: np.nanstd(a, ddof=1) * np.sqrt(252) * 100)
```
注意：`np.nanstd` 在全 NaN 窗口返回 NaN 并抛 RuntimeWarning，应加 `warnings.catch_warnings` 或显式检查。

### P1-B: `_feature_bbwidth` 把 NaN middle 替换为 0，掩盖缺失数据

**文件:行号**: [functions.py:1278-1282](file:///d:/01-Doc/Quant/ft2/utils/ast/functions.py#L1278-L1282)

**问题描述**:
```python
def _feature_bbwidth(close, period=20):
    c = np.asarray(close, float)
    upper, middle, lower = talib.BBANDS(c, timeperiod=period, nbdevup=2, nbdevdn=2, matype=0)

```

`np.where(middle > 0, x, 0)`：当 middle 是 NaN 时，`NaN > 0` 为 False，返回 0。停牌期的 bb_width 变成 0，被下游当作"极窄带"信号。

**修复方案**:

```python
# 显式区分 "middle<=0 (异常)" 和 "middle=NaN (缺失)"
mask_valid = np.isfinite(middle) & (middle > 0)
return np.where(mask_valid, (upper - lower) / np.where(mask_valid, middle, 1.0), np.nan)
```

### P1-D: `cross_sectional_rank` 与 `_cs_rank_core` 全 NaN 行行为不一致

**文件:行号**: [dsl.py:508-528](file:///d:/01-Doc/Quant/ft2/utils/ast/dsl.py#L508-L528) vs [functions.py:887-919](file:///d:/01-Doc/Quant/ft2/utils/ast/functions.py#L887-L919)

**问题描述**:

| 实现 | 全 NaN 行输出 | 备注 |
|------|:---:|------|
| `cross_sectional_rank` (dsl.py) | `0.0` | `ranked[t, valid]` 不赋值，保持初始 0.0 |
| `_cs_rank_core` (functions.py) | `NaN` | `out = np.full(..., np.nan)`，`cnt==0` 时 continue |

两份代码处理同一语义（截面排名），全 NaN 行返回值不一致。

**影响分析**:
- `CsResolver.resolve()` 末尾调用 `cross_sectional_rank(vals)` 做最终排名（[resolver.py:93](file:///d:/01-Doc/Quant/ft2/utils/ast/resolver.py#L93), [resolver.py:105](file:///d:/01-Doc/Quant/ft2/utils/ast/resolver.py#L105)）。

- 外层 `cross_sectional_rank` 把含 NaN 的行用 0.0 填充，**覆盖了内层 NaN 标记**。
- 最终结果：某日某列表达式值为 NaN（缺失），但 cross_sectional_rank 给出 0.0 排名，下游 `cs_rank(ts_rank(cs_rank(A), 5))` 会把这个 0.0 当真实排名参与时间排名。

**修复方案**: 统一为 NaN 语义：
```python
def cross_sectional_rank(vals):
    # ...
    for t in range(T):
        row = vals[t]
        valid = ~np.isnan(row)
        if valid.sum() > 0:
            rk = rankdata(row[valid], method='min')
            ranked[t, valid] = rk / valid.sum()
        # 全 NaN 行保持 NaN（不再填充 0.0）
    return ranked
```
或更彻底：直接调用 `_cs_rank_core`（numba 加速 + 语义统一）。

### 已修复 (第一轮 6 项 + 本轮无新增修复)
核心 numba 改造和 P0/P1 bug 已全部修复，回归测试通过。

### 待处理 (本轮新发现 4 项 P1 + 5 项 P2 + 3 项 P3)

**关键风险**: P1-A/B/C 是同一类问题——特征函数用 `np.where(x > 0, ..., 0)` 模式处理 NaN，把"缺失"伪装成"极值"。与第一轮 P0-1（ts_rank 过期排名）是同类语义错误，会在 A 股停牌数据上静默污染因子值。

**P1-D 是跨文件一致性问题**: `cross_sectional_rank`（dsl.py）和 `_cs_rank_core`（functions.py）是两份独立实现的截面排名，全 NaN 行行为不一致。CsResolver 同时使用两者，存在语义裂缝。

### 修复优先级建议

1. **优先修复 P1-A/B/C**（特征函数 NaN 处理）：3 处 `np.where` 改为显式 NaN 检查，工作量小、风险低、收益高。
2. **统一 P1-D**：`cross_sectional_rank` 改为调用 `_cs_rank_core`，消除代码重复 + 语义统一。
3. **P2-B/C**：文档对齐，1 行修改。
4. **P2-D**：3 个函数加 `_validate_window`，1 行修改。
5. **P2-A/P3**：低优先级，可后续处理。
